The Problem

AI agents drift. OpenContext doesn't let them.

When you give a Jira ticket to an AI agent with no context, it guesses your stack, invents patterns, and ships code that looks right but breaks everything. OpenContext fixes this by making the agent's context ground truth.

✗

Without OpenContext

  • Agent invents libraries you don't use
  • New code ignores existing patterns
  • Jira ticket context is lost by message 3
  • Architecture decisions get repeated from scratch
  • No guardrails — agent can suggest anything
✓

With OpenContext

  • Spec & Jira facts vectorized into RAG store
  • Agent task anchored to architectural constraints
  • Git diff insights injected at every commit
  • Zero Drift — forbidden tech explicitly blocked
  • Repeatable, consistent, architecture-safe output
Workflow

Four commands. Complete architecture safety.

01
🔑

opencontext setup

One-time global configuration. Stores your Jira email & API token in ~/.opencontext/global.config.json and installs a Git pre-commit hook that auto-captures staged diffs.

Global
↓
02
📐

opencontext init

Project-level init. Sets your Jira domain in opencontext.config.json, then fetches all OpenSpec design documents and converts them into a structured context for the AI agent.

Per Project
↓
03
🧠

opencontext initbranch

Run after git checkout -b <ticket>. Vectorizes architectural facts with sentence-transformers + LanceDB, fetches the Jira ticket, and generates a fully-grounded agent.md task.

Per Branch
↓
04
🔄

opencontext sync

Re-runs during development. Re-vectorizes the latest diff_analysis.json (captured by the Git hook) and regenerates the agent task — keeping context fresh with every commit.

Architecture

Under the hood

📡

Fetching Layer

Pulls live data from Jira REST API, OpenSpec markdown files, and Git staged diffs. All raw context feeding into the pipeline.

jira_retriever.py spec_fetcher.py git_diff.py
🔢

Pipeline Layer

Converts text into dense vector embeddings using all-MiniLM-L6-v2 via sentence-transformers, stored in a local LanceDB vector database.

vectorizer.py LanceDB sentence-transformers
⚡

Inference Layer

RAG-retrieves the top-3 relevant context chunks matching the Jira ticket, then assembles a structured agent.md task with Zero Drift guardrails.

inference/main.py RAG retrieval agent.md
🖥️

CLI Layer

Typer-based CLI exposing 4 commands. Git hook auto-triggers diff capture on every git commit, keeping context always current.

cli/main.py Typer pre-commit hook
